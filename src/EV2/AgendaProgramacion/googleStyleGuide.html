Práctica sobre la Guía de Estilo de Java de Google

  1. Los ficheros ".java

    • El nombre de los ficheros .java debe ser el mismo que el nombre de la clase que contiene, en upper CamelCase. 2.1

    • La codificación tiene que ser UTF-8. 2.2

    • No se usa TAB para indentar bloques de código. 2.3.1

    • Se usan solo las secuencias de escape de java: \b, \t, \n,\f, \r,
      \", \', \\. 2.3.2

    • Para caracteres no ASCII se utiliza el propio caracter antes que el código unicode. 2.3.3

    • La estructura de los ficheros .java es la siguiente:

      • Licencia, si la hubiera. 3.1

      • Package. 3.2

      • Imports. 3.3

      • Declaración de la clase. Los métodos sobrecargados NO se separan. 3.4

    • Límite de columnas: 100. 4.4
